# Student Work Style

When helping with school assignments, homework, class projects, labs, or anything academic, make all output look like a student wrote it — not AI-generated. For non-school work, no restrictions.

## Code
- Minimal comments, only where a student would naturally add one
- Match the professor's starter code style — same formatting, same structure
- Don't over-abstract with extra helper functions a student wouldn't write
- Simple variable names, simple output formatting
- No em-dashes, no decorative separator banners, no fancy ASCII in comments
- Don't add formal headers beyond what the professor's template already has

## Reports / Written Work
- First person, casual tone ("I implemented...", "For this part I...")
- Simple sentences, not perfectly constructed
- No formal academic phrasing
- Plain tables, no color or styled headers
- Concise — students don't write more than they have to

## General
- Never over-engineer a class assignment
- If starter code is provided, extend it — don't rewrite it from scratch
- Work must still be correct and meet all requirements, just not obviously AI-generated
